ES6語法糖 - 箭頭函式與this(下) - call、apply、bind

發表於 2020-11-02 更新於 2020-11-04 分類於 JavaScript
this與內建函式 call、apply、bind 如何運作?

函式的內建方法與關鍵字

在箭頭函式與 this(上)中提到，箭頭函式沒有這些方式可以用 :

  • this
  • arguments
  • call()
  • apply()
  • bind()

這篇我們要來看這些東西原本是怎麼運作的。

關鍵字 this

複習一下 this 的運作 :

關鍵字 arguments

  • 一般函式內建的關鍵字之一
  • 可以將傳入的參數值抓取出來形成一個類陣列物件

內建函式 call、apply、bind的共通點

  • JS的內建函式
  • 它們都與 this 有所關聯
  • 不需要在函式中自訂參數就能將變數塞進 this
  • 若有多個值，函式的第一個參數位置為 this
  • 也就是說(this, 參數1, 參數2, 以此類推)

接下來，我們來看，call()、apply()以及 bind() 如何與 this 運作。

內建函式 call

  • 單純將參數值傳入

內建函式 apply

  • 傳入的型態必須是陣列
  • 功能上與 call 無異

內建函式 bind

  • bind() 將建立一個新的函式

箭頭函式與 this

我們了解到箭頭函式沒有自己的 this，相對的也就沒有上述的那三種內建函式可供使用，也無法使用 arguments，但可以改用其餘參數來達成目的。箭頭函式與一般函式的運用，還是得看實際情況來做使用。
